需求一
a、平台功能
1、系统管理：用户、角色、菜单+按钮、组织树、登录账户、密码、创建时间、创建人
2、店铺管理：开店铺，包括基础信息+开通店铺管理员账户、创建时间、创建人
3、商品管理：
3.1:商品分类：序号、分类名称、价格、排序、创建时间、创建人
3.2:商品列表：序号、编号、名称、排序、流程配置（因为我大部分都是服务性商品，所有有跟进流程的，比如，联系用户、下单、创建账户、部署、实施等节点，每个商品可以设置固定的节点流程）、创建人、创建时间、关联线上商城商品编码（可选、默认为空，做保留字段前端默认不展示）

4、财务管理：
4.1、收支分类：三级（收入-营业收入-服务性收入）、序号、名称、排序、创建人、创建时间、财务编号（默认为空）
4.2 收支报表、收入支出、利润

需求二、
b、店铺功能：所有数据以店铺纬度展示
1、系统管理：用户、角色、菜单+按钮、组织树、登录账户、密码、创建时间、创建人
2、商品管理：
1）从平台选择商品，然后存到自己店铺的表中，相当于这个店铺可以卖多少个商品，只能从平台那边的标准商品库选择几个，流程复用平台的设置，不允许自行设置，价格可以自己设置，默认与平台一致、创建人、创建时间

3、客户管理：序号、类型（个人、企业）、名称、联系人、联系电话、创建人、创建时间

4、订单管理：序号、选择客户、选择商品（可以多选、但是生成多个订单，单号一致一个商品一条，因为不同的商品服务流程不同）、数量（一个商品对应一个数量）、展示价格（不可编辑）、创建人、创建时间、订单号自动生成（时间戳）、跟进时每一步可以填写服务备注、可以选择上传附件、可以查看所有的跟进记录

5、财务管理：
1）设置收支账户：类型（对公账户、微信、支付宝）、账号、开户行、初始余额、微信等对应的就是商户号、appid等
2）收支分类：从平台选择收支分类，存到自己的表中，只能选择不能新增和编辑，、创建人、创建时间
3）收支列表：选择账户、选择收支类型（收入或者支出，三级）、金额、审核（通过、驳回）、审核可上传附件、填写真实入账金额、关联订单号（可选）
4）财务报表

性能需求：要求高可用、高并发、高安全、高可靠、高性能
